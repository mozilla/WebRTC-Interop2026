## Executive Summary

The `interop-2026-webrtc` tracked test list was significantly pruned this week, dropping from 101 to 90 scored tests. Eleven of the twelve removed tests came from `webrtc-encoded-transform/` as that suite was re-scoped around the merged SFrame API renames ([PR#308](https://github.com/w3c/webrtc-encoded-transform/pull/308), [PR#309](https://github.com/w3c/webrtc-encoded-transform/pull/309)); the other notable removal was `webrtc/protocol/h264-profile-levels.https.html?interop-2026`, which had been under discussion for removal since April. As a result, Firefox's aggregate failing count dropped from 92 to **87** and single-browser failures from 90 to **85** — entirely from scope refinement, with no Firefox code changes affecting tracked-test scores. In addition: bug [2019378](https://bugzilla.mozilla.org/show_bug.cgi?id=2019378) (RTCSctpTransport-maxChannels) remains RESOLVED with its WPT companion [PR#59875](https://github.com/web-platform-tests/wpt/pull/59875) in flight; [1814461](https://bugzilla.mozilla.org/show_bug.cgi?id=1814461) (RTCError for DTLS failures) was duped into [1805447](https://bugzilla.mozilla.org/show_bug.cgi?id=1805447) (RTCDtlsTransport.onerror); and Byron Campen filed two new W3C spec issues ([webrtc-pc#3109](https://github.com/w3c/webrtc-pc/issues/3109), [#3110](https://github.com/w3c/webrtc-pc/issues/3110)). Active engineering continues on the RTCTransportStats and RTCIceTransport clusters with no merged code landings this run.
